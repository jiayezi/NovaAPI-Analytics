# NovaAPI AI API 聚合平台指标计算指南

本指南旨在为 NovaAPI-Analytics 项目的指标计算提供统一的逻辑定义与实现参考。文档重点保留了具有高度代表性、避免重复计算逻辑的核心指标。

---

## 1. 平台视图 (Platform View) - 全局业务看板

### A. 用户增长与留存 (Growth & Retention)
| 指标名称 | 定义 | 计算难点 / 练习点 | 业务价值 |
| :--- | :--- | :--- | :--- |
| **MAU (月活跃用户)** | 每月至少有一次成功 API 调用请求的用户总数。 | 需要对 `fct_account_daily_snapshot` 进行去重统计。 | 衡量平台的基础用户规模。 |
| **留存率 (Cohort Analysis)** | 以月为单位，观察某一注册月份的用户在后续月份的活跃比例。 | 复杂的 SQL 自连接或 Pandas `groupby` 转换。 | 判断产品粘性与长期价值。 |
| **订阅流转详情 (SCD2)** | 每月新注册、升级（如 Free -> Pro）、降级、流失用户的数量及环比。 | **重点**：需关联 `user_plan_changes` 判定状态变更。 | 验证商业化漏斗驱动力，体现 SCD2 设计价值。 |

### B. 财务健康度 (Financials)
| 指标名称 | 定义 | 计算难点 / 练习点 | 业务价值 |
| :--- | :--- | :--- | :--- |
| **Monthly Profit (月毛利)** | `总充值金额 - 对应的模型调用成本`。 | 需将 Token 消耗乘以 `ai_models` 表中的成本单价。 | 衡量平台真实的盈利能力。 |
| **ARPU (每用户平均收入)** | 总营收 / 总活跃用户数。 | 跨表聚合计算。 | 评估单个用户的商业贡献。 |
| **利润预测 (Forecasting)** | 基于过去 90 天的利润数据，预测下月利润走势。 | 使用 Prophet 或 ARIMA 等时间序列模型。 | 为业务扩张提供前瞻性参考。 |

### C. 模型表现与核心质量 (QoS)
| 指标名称 | 定义 | 计算难点 / 练习点 | 业务价值 |
| :--- | :--- | :--- | :--- |
| **1M Tokens 利润贡献度** | 衡量各模型每消耗 100 万个 Token 能产生的净利润。 | 需要对各模型的计费与成本进行归一化。 | **核心优选**：用于决策哪些模型该主推。 |
| **P99 Latency & 错误率** | 剔除异常波动后的 99 分位延迟以及 HTTP 非 200 占比。 | 窗口函数 `PERCENTILE_CONT` 的应用。 | 监控系统性能，捕捉模拟数据中的模型抖动。 |

### D. 异常与画像 (Advanced)
| 指标名称 | 定义 | 计算难点 / 练习点 | 业务价值 |
| :--- | :--- | :--- | :--- |
| **实时异常告警数** | 被统计模型（如 3-sigma）识别出的流量爆发或延迟突刺事件。 | 识别 `main.py` 中注入的“黑客刷单”行为。 | 验证异常检测算法的准确性。 |
| **用户价值分层 (RFM)** | 基于活跃频率、消费金额将用户聚类为 4-5 个群体。 |应用 K-Means 聚类或简单的规则打分。 | 进行差异化运营（如对高价值用户启用白名单）。 |

## 2. 用户视图 (User View) - 个人仪表盘

| 指标名称 | 定义 | 练习点 |
| :--- | :--- | :--- |
| **消费趋势追踪** | 每日的调用次数、Token 消耗量与账户扣款金额。 | 散点图 + 趋势折线图数据聚合。 |
| **Key 级消耗排行榜** | 列出当前用户名下各个 API Key 的使用情况。 | 辅助用户清理无效 Key，降低账户风险。 |
| **偏好画像** | 标识出用户最常使用的模型 Top 3 及其对应的平均响应时间。 | 用户端的个性化推荐基础。 |

## 3. 指标开发 Roadmap (分阶段建议)

### 第一阶段：基础看板 (Focus on CRUD & Aggregation)
- [ ] MAU 统计与模型活跃分布。
- [ ] 基于 `billing_orders` 的基础利润计算。

### 第二阶段：深度分析 (Focus on Time & Logic)
- [ ] 留存率队列分析 (Cohort Analysis)。
- [ ] 基于 `user_plan_changes` 的订阅流转分析。
- [ ] P99 延迟与错误率异常监控。

### 第三阶段：AI 增强 (Focus on ML & Sci-Math)
- [ ] 用户价值 RFM 聚类模型。
- [ ] 下月盈利预测模型。
- [ ] 自动化异常检测（定位黑客用户）。

---

## 4. 核心计算规则注意
1. **统一单位**：所有金额统一保留 4 位小数，Tokens 统一转化为 "百万 (1M)" 单位进行对比。
2. **时间对齐**：指标计算时，请确保 `request_time` (日志时间) 与 `created_at` (订单时间) 的时区一致性。
3. **性能优化**：对于 `request_logs_raw` 等大表，建议使用增量聚合（Incremental Aggregation）而非全表扫描。
